# Estima
## Итоговый проект по второй части курса DLS (МФТИ ФПМИ)
Идея реализовать AI-ассистента для оценки реализации ИТ проекта.

## Краткое описание
Пользователь через UI загружает файл (PDF, TXT, ODT, DOCX) с ТЗ или брифом от потенциального заказчика. Дополнительно указывает настройки.

Система сохраняет файл и считывает контент файла.

С помощью LLM происходит выделение этапов работ, задач, ролей специалистов (backend, frontend, аналитик, архитектор, qa, pm, devops), если пользователь не указал роли в настройках.

Для каждой выделенной задачи находим дополнительные знания (прошлые оценки, лучшие практики) в векторной БД.

Формируем итоговый запрос к LLM для оценки проекта. С помощью трёх разных LLM происходит оценка задач в человеко-часах, выделенных на первом шаге.

После этого усредняем оценки и возвращаем итоговый результат пользователю. На выходе Excel или csv файл.

## Базовая последовательность действий
1. Пользователь загружает файл и задает настройки.
2. Сохраняем файл.
3. Считываем контент документа из файла.
4. С помощью LLM анализируем текст документа, чтобы выделить этапы работ и в каждом этапе задачи. LLM возвращает JSON с этапами и задачами.
4.1 Если пользователь не указал роли сам, то дополнительно LLM с этапами и задачами должна вернуть набор ролей сотрудников, которые необходимы для реализации проекта.
5. С помощью LLM создаем краткое содержание исходного документа. LLM возвращает только текст.
6. Для каждой задачи, которую выделила LLM, осуществляем поиск дополнительных знаний (оценка в человеко-часах и другие метаданные) в векторной БД.
7. Формируем итоговый запрос к LLM для оценки проекта из: краткого содержания исходного документа, переченя этапов работ и задач, переченя ролей, дополнительных знаний из векторной БД.
8. Выполняем три запроса к разным LLM для оценки работ.
9. Собираем оценки от трех моделей и усредняем значения.
10. Оформляем итоговый результат в Excel файл.
